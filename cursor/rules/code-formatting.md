# Code Formatting Rules

## Indentation
- Use 2 spaces for indentation across all file types
- No tab characters allowed in code

## Line Length
- Maximum line length is 100 characters
- Break long lines at logical points (operators, commas, etc.)

## Whitespace
- No trailing whitespace
- One empty line at the end of each file
- Use empty lines to group related code sections
- No multiple empty lines in a row

## Semicolons
- Always use semicolons in JavaScript/TypeScript

## Quotes
- Use single quotes (`'`) for JavaScript/TypeScript strings
- Use double quotes (`"`) for JSX/HTML attributes

## Braces
- Opening brace on the same line as the statement
- Closing brace on its own line
- Always use braces for control structures, even for single-line blocks
